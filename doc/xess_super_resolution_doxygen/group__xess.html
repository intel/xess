◆ xessForceLegacyScaleFactors()

XESS_API xess_result_t xessForceLegacyScaleFactors ( xess_context_handle_t  hContext,
bool  force 
)

Forces usage of legacy (pre 1.3.0) scale factors.

Following scale factors will be applied:

  • XESS_QUALITY_SETTING_ULTRA_PERFORMANCE: 3.0
  • XESS_QUALITY_SETTING_PERFORMANCE: 2.0
  • XESS_QUALITY_SETTING_BALANCED: 1.7
  • XESS_QUALITY_SETTING_QUALITY: 1.5
  • XESS_QUALITY_SETTING_ULTRA_QUALITY: 1.3
  • XESS_QUALITY_SETTING_AA: 1.0 In order to apply new scale factors application should call xessGetOptimalInputResolution and initialization function (xess*Init)
Parameters
hContextThe XeSS context handle.
forceif set to true legacy scale factors will be forced, if set to false - scale factors will be selected by XeSS
Returns
XeSS return status code.

◆ xessGetOptimalInputResolution()

XESS_API xess_result_t xessGetOptimalInputResolution ( xess_context_handle_t  hContext,
const xess_2d_t *  pOutputResolution,
xess_quality_settings_t  qualitySettings,
xess_2d_t *  pInputResolutionOptimal,
xess_2d_t *  pInputResolutionMin,
xess_2d_t *  pInputResolutionMax 
)

Get the optimal input resolution and possible range for a specified output resolution for a given quality setting. XeSS expects all the input buffers except motion vectors to be in the returned resolution range and all input buffers to be in the same resolution. Motion vectors can be either in output resolution (XESS_INIT_FLAG_HIGH_RES_MV) or in the same resolution as other input buffers (by default).

Note
Aspect ratio of the input resolution must be the same as for the output resolution.
Parameters
hContextThe XeSS context handle.
pOutputResolutionOutput resolution to calculate input resolution for.
qualitySettingsDesired quality setting.
[out]pInputResolutionOptimalOptimal input resolution.
[out]pInputResolutionMinRequired minimal input resolution.
[out]pInputResolutionMaxRequired maximal input resolution.
Returns
XeSS return status code.

◆ xessGetPipelineBuildStatus()

XESS_API xess_result_t xessGetPipelineBuildStatus ( xess_context_handle_t  hContext)

Returns current state of pipeline build This function can only be called after xess*BuildPipelines and before corresponding xess*Init. This call returns XESS_RESULT_SUCCESS if pipelines already built, and XESS_RESULT_ERROR_OPERATION_IN_PROGRESS if pipline build is in progress. If function called before XeSS API exports*BuildPipelines or after XeSS API exports*Init - XESS_RESULT_ERROR_WRONG_CALL_ORDER will be returned.

Parameters
hContextThe XeSS context handle.
Returns
XESS_RESULT_SUCCESS if pipelines already built. XESS_RESULT_ERROR_OPERATION_IN_PROGRESS if pipeline build are in progress. XESS_RESULT_ERROR_WRONG_CALL_ORDER if the function is called out of order.
